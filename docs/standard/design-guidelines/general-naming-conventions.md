# <a name="general-naming-conventions"></a><span data-ttu-id="8c546-102">Convenzioni di denominazione generali</span><span class="sxs-lookup"><span data-stu-id="8c546-102">General Naming Conventions</span></span>
<span data-ttu-id="8c546-103">In questa sezione vengono descritte le convenzioni di denominazione generali che riguardano la scelta di Word, le linee guida sull'utilizzo delle abbreviazioni e degli acronimi e consigli su come evitare l'utilizzo di nomi specifici della lingua.</span><span class="sxs-lookup"><span data-stu-id="8c546-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>

## <a name="word-choice"></a><span data-ttu-id="8c546-104">Scelta di Word</span><span class="sxs-lookup"><span data-stu-id="8c546-104">Word Choice</span></span>
 <span data-ttu-id="8c546-105">✔️ scegliere nomi di identificatori facilmente leggibili.</span><span class="sxs-lookup"><span data-stu-id="8c546-105">✔️ DO choose easily readable identifier names.</span></span>

 <span data-ttu-id="8c546-106">Una proprietà denominata `HorizontalAlignment`, ad esempio, è più leggibile in inglese rispetto a `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="8c546-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>

 <span data-ttu-id="8c546-107">✔️ prediligono la leggibilità per brevità.</span><span class="sxs-lookup"><span data-stu-id="8c546-107">✔️ DO favor readability over brevity.</span></span>

 <span data-ttu-id="8c546-108">Il nome della proprietà `CanScrollHorizontally` è migliore di `ScrollableX` (un riferimento oscuro all'asse X).</span><span class="sxs-lookup"><span data-stu-id="8c546-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>

 <span data-ttu-id="8c546-109">❌ non usare caratteri di sottolineatura, trattini o altri caratteri non alfanumerici.</span><span class="sxs-lookup"><span data-stu-id="8c546-109">❌ DO NOT use underscores, hyphens, or any other nonalphanumeric characters.</span></span>

 <span data-ttu-id="8c546-110">❌ non utilizzano la notazione ungherese.</span><span class="sxs-lookup"><span data-stu-id="8c546-110">❌ DO NOT use Hungarian notation.</span></span>

 <span data-ttu-id="8c546-111">❌ evitare l'utilizzo di identificatori in conflitto con parole chiave di linguaggi di programmazione ampiamente utilizzati.</span><span class="sxs-lookup"><span data-stu-id="8c546-111">❌ AVOID using identifiers that conflict with keywords of widely used programming languages.</span></span>

 <span data-ttu-id="8c546-112">In base alla regola 4 della Common Language Specification (CLS), tutti i linguaggi conformi devono fornire un meccanismo che consenta l'accesso agli elementi denominati che usano una parola chiave di tale lingua come identificatore.</span><span class="sxs-lookup"><span data-stu-id="8c546-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="8c546-113">C#, ad esempio, in questo caso usa il segno @ come meccanismo di escape.</span><span class="sxs-lookup"><span data-stu-id="8c546-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="8c546-114">Tuttavia, è comunque consigliabile evitare le parole chiave comuni perché è molto più difficile usare un metodo con la sequenza di escape rispetto a una senza di essa.</span><span class="sxs-lookup"><span data-stu-id="8c546-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>

## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="8c546-115">Uso delle abbreviazioni e degli acronimi</span><span class="sxs-lookup"><span data-stu-id="8c546-115">Using Abbreviations and Acronyms</span></span>
 <span data-ttu-id="8c546-116">❌ non utilizzano abbreviazioni o contrazioni come parte dei nomi degli identificatori.</span><span class="sxs-lookup"><span data-stu-id="8c546-116">❌ DO NOT use abbreviations or contractions as part of identifier names.</span></span>

 <span data-ttu-id="8c546-117">Ad esempio, usare `GetWindow` anziché `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="8c546-117">For example, use `GetWindow` rather than `GetWin`.</span></span>

 <span data-ttu-id="8c546-118">❌ non utilizzano alcun acronimo che non sia ampiamente accettato e anche se sono, solo quando necessario.</span><span class="sxs-lookup"><span data-stu-id="8c546-118">❌ DO NOT use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="8c546-122">✔️ utilizzano un nome di tipo CLR generico, anziché un nome specifico della lingua, nei rari casi in cui un identificatore non ha un significato semantico oltre il relativo tipo.</span><span class="sxs-lookup"><span data-stu-id="8c546-122">✔️ DO use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>
