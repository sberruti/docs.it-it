# <a name="general-naming-conventions"></a><span data-ttu-id="54744-102">Convenzioni di denominazione generali</span><span class="sxs-lookup"><span data-stu-id="54744-102">General Naming Conventions</span></span>
<span data-ttu-id="54744-103">In questa sezione vengono descritte le convenzioni di denominazione generali che riguardano la scelta di Word, le linee guida sull'utilizzo delle abbreviazioni e degli acronimi e consigli su come evitare l'utilizzo di nomi specifici della lingua.</span><span class="sxs-lookup"><span data-stu-id="54744-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>  
  
## <a name="word-choice"></a><span data-ttu-id="54744-104">Scelta di Word</span><span class="sxs-lookup"><span data-stu-id="54744-104">Word Choice</span></span>  
 <span data-ttu-id="54744-105">**✓ DO** scegliere i nomi degli identificatori facilmente leggibili.</span><span class="sxs-lookup"><span data-stu-id="54744-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="54744-106">Una proprietà denominata `HorizontalAlignment`, ad esempio, è più leggibile in inglese rispetto a `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="54744-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="54744-107">**✓ DO** privilegiare la leggibilità ragioni di brevità.</span><span class="sxs-lookup"><span data-stu-id="54744-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="54744-108">Il nome della proprietà `CanScrollHorizontally` è migliore di `ScrollableX` (un riferimento oscuro all'asse X).</span><span class="sxs-lookup"><span data-stu-id="54744-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="54744-109">**X DO NOT** utilizzare caratteri di sottolineatura, trattini o altri caratteri non alfanumerici.</span><span class="sxs-lookup"><span data-stu-id="54744-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="54744-110">**X DO NOT** utilizzare la notazione ungherese.</span><span class="sxs-lookup"><span data-stu-id="54744-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="54744-111">**X AVOID** utilizzando gli identificatori che sono in conflitto con le parole chiave di ampiamente utilizzato linguaggi di programmazione.</span><span class="sxs-lookup"><span data-stu-id="54744-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="54744-112">In base alla regola 4 della Common Language Specification (CLS), tutti i linguaggi conformi devono fornire un meccanismo che consenta l'accesso agli elementi denominati che usano una parola chiave di tale lingua come identificatore.</span><span class="sxs-lookup"><span data-stu-id="54744-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="54744-113">C#, ad esempio, in questo caso usa il segno @ come meccanismo di escape.</span><span class="sxs-lookup"><span data-stu-id="54744-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="54744-114">Tuttavia, è comunque consigliabile evitare le parole chiave comuni perché è molto più difficile usare un metodo con la sequenza di escape rispetto a una senza di essa.</span><span class="sxs-lookup"><span data-stu-id="54744-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="54744-115">Uso delle abbreviazioni e degli acronimi</span><span class="sxs-lookup"><span data-stu-id="54744-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="54744-116">**X DO NOT** utilizzare abbreviazioni o termini più semplici come parte dei nomi degli identificatori.</span><span class="sxs-lookup"><span data-stu-id="54744-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="54744-117">Ad esempio, usare `GetWindow` anziché `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="54744-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="54744-118">**X DO NOT** utilizzare acronimi che non sono molto diffusa e anche se sono solo quando necessario.</span><span class="sxs-lookup"><span data-stu-id="54744-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="54744-120">**✓ DO** utilizzare nomi significativi anziché le parole chiave specifiche della lingua per i nomi dei tipi.</span><span class="sxs-lookup"><span data-stu-id="54744-120">**✓ DO** use semantically interesting names rather than language-specific keywords for type names.</span></span>  
  
 <span data-ttu-id="54744-121">Ad esempio, `GetLength` è un nome migliore rispetto a `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="54744-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="54744-122">**✓ DO** usare un nome di tipo generico di CLR, anziché un nome specifico della lingua, in rari casi in cui un identificatore non ha alcun significato semantico oltre il relativo tipo.</span><span class="sxs-lookup"><span data-stu-id="54744-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

## <a name="naming-new-versions-of-existing-apis"></a><span data-ttu-id="54744-186">Assegnazione di un nome alle nuove versioni delle API esistenti</span><span class="sxs-lookup"><span data-stu-id="54744-186">Naming New Versions of Existing APIs</span></span>  
 <span data-ttu-id="54744-187">**✓ DO** utilizzare un nome simile all'API precedente durante la creazione di nuove versioni di un'API esistente.</span><span class="sxs-lookup"><span data-stu-id="54744-187">**✓ DO** use a name similar to the old API when creating new versions of an existing API.</span></span>  
  
 <span data-ttu-id="54744-188">Ciò consente di evidenziare la relazione tra le API.</span><span class="sxs-lookup"><span data-stu-id="54744-188">This helps to highlight the relationship between the APIs.</span></span>  
  
 <span data-ttu-id="54744-189">**✓ DO** preferisce aggiunta di un suffisso anziché un prefisso per indicare una nuova versione di un'API esistente.</span><span class="sxs-lookup"><span data-stu-id="54744-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="54744-190">Questa operazione aiuterà l'individuazione durante l'esplorazione della documentazione o l'uso di IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="54744-190">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="54744-191">La versione precedente dell'API verrà organizzata vicino alle nuove API, perché la maggior parte dei browser e di IntelliSense mostrano gli identificatori in ordine alfabetico.</span><span class="sxs-lookup"><span data-stu-id="54744-191">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>

 <span data-ttu-id="54744-193">**✓ DO** utilizzare un suffisso numerico per indicare una nuova versione di un'API esistente, in particolare se il nome dell'API esistente è l'unico nome appropriato (ad esempio, se è uno standard del settore) e se si aggiungono qualsiasi significativo suffisso (o la modifica del nome) non è un'app opzione ropriate.</span><span class="sxs-lookup"><span data-stu-id="54744-193">**✓ DO** use a numeric suffix to indicate a new version of an existing API, particularly if the existing name of the API is the only name that makes sense (i.e., if it is an industry standard) and if adding any meaningful suffix (or changing the name) is not an appropriate option.</span></span>
