<span data-ttu-id="aa17c-103">Un nome sicuro è costituito dall'identità dell'assembly, corrispondente al nome semplice in formato testo, al numero di versione e alle informazioni sulle impostazioni cultura eventualmente disponibili, nonché da una chiave pubblica e da una firma digitale.</span><span class="sxs-lookup"><span data-stu-id="aa17c-103">A strong name consists of the assembly's identity—its simple text name, version number, and culture information (if provided)—plus a public key and a digital signature.</span></span> <span data-ttu-id="aa17c-104">Questo nome viene generato da un file di assembly usando la chiave privata corrispondente.</span><span class="sxs-lookup"><span data-stu-id="aa17c-104">It is generated from an assembly file using the corresponding private key.</span></span> <span data-ttu-id="aa17c-105">Il file di assembly contiene il manifesto dell'assembly, che include i nomi e gli hash di tutti i file che costituiscono l'assembly.</span><span class="sxs-lookup"><span data-stu-id="aa17c-105">(The assembly file contains the assembly manifest, which contains the names and hashes of all the files that make up the assembly.)</span></span>

## <a name="strong-name-scenario"></a><span data-ttu-id="aa17c-112">Scenario con nome sicuro</span><span class="sxs-lookup"><span data-stu-id="aa17c-112">Strong name scenario</span></span>

<span data-ttu-id="aa17c-113">Lo scenario seguente descrive il processo di firma di un assembly con un nome sicuro per farvi successivamente riferimento in base a tale nome.</span><span class="sxs-lookup"><span data-stu-id="aa17c-113">The following scenario outlines the process of signing an assembly with a strong name and later referencing it by that name.</span></span>

1. <span data-ttu-id="aa17c-114">L'assembly A viene creato con un nome sicuro usando uno dei metodi seguenti:</span><span class="sxs-lookup"><span data-stu-id="aa17c-114">Assembly A is created with a strong name using one of the following methods:</span></span>

    - <span data-ttu-id="aa17c-115">Usando un ambiente di sviluppo che supporta la creazione di nomi sicuri, come Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="aa17c-115">Using a development environment that supports creating strong names, such as Visual Studio.</span></span>

    - <span data-ttu-id="aa17c-116">Creando una coppia di chiavi crittografiche usando lo [strumento Nome sicuro (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) e assegnandola all'assembly mediante un compilatore della riga di comando o [Assembly Linker (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span><span class="sxs-lookup"><span data-stu-id="aa17c-116">Creating a cryptographic key pair using the [Strong Name tool (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) and assigning that key pair to the assembly using either a command-line compiler or the [Assembly Linker (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span></span> <span data-ttu-id="aa17c-117">Windows SDK fornisce sia Sn.exe che Al.exe.</span><span class="sxs-lookup"><span data-stu-id="aa17c-117">The Windows SDK provides both Sn.exe and Al.exe.</span></span>

2. <span data-ttu-id="aa17c-118">L'ambiente di sviluppo o lo strumento firma l'hash del file contenente il manifesto dell'assembly con la chiave privata dello sviluppatore.</span><span class="sxs-lookup"><span data-stu-id="aa17c-118">The development environment or tool signs the hash of the file containing the assembly's manifest with the developer's private key.</span></span> <span data-ttu-id="aa17c-119">La firma digitale viene archiviata nel file eseguibile portabile (PE) che contiene il manifesto dell'assembly A.</span><span class="sxs-lookup"><span data-stu-id="aa17c-119">This digital signature is stored in the portable executable (PE) file that contains Assembly A's manifest.</span></span>

3. <span data-ttu-id="aa17c-120">L'assembly B è un consumer dell'assembly A. La sezione di riferimento del manifesto dell'assembly B include un token che rappresenta la chiave pubblica dell'assembly A.</span><span class="sxs-lookup"><span data-stu-id="aa17c-120">Assembly B is a consumer of Assembly A. The reference section of Assembly B's manifest includes a token that represents Assembly A's public key.</span></span> <span data-ttu-id="aa17c-121">Un token è una parte della chiave pubblica completa e viene usato al posto della chiave stessa per risparmiare spazio.</span><span class="sxs-lookup"><span data-stu-id="aa17c-121">A token is a portion of the full public key and is used rather than the key itself to save space.</span></span>

4. <span data-ttu-id="aa17c-122">Common Language Runtime verifica la firma con nome sicuro quando l'assembly viene inserito nella Global Assembly Cache.</span><span class="sxs-lookup"><span data-stu-id="aa17c-122">The common language runtime verifies the strong name signature when the assembly is placed in the global assembly cache.</span></span> <span data-ttu-id="aa17c-123">Quando si esegue l'associazione in base al nome sicuro in runtime, Common Language Runtime confronta la chiave archiviata nel manifesto dell'assembly B con la chiave usata per generare il nome sicuro per l'assembly A. Se i controlli di sicurezza di .NET Framework vengono superati e l'associazione riesce, l'assembly B ha la garanzia che i bit dell'assembly A non siano stati manomessi e che questi bit provengono in effetti dagli sviluppatori dell'assembly A.</span><span class="sxs-lookup"><span data-stu-id="aa17c-123">When binding by strong name at run time, the common language runtime compares the key stored in Assembly B's manifest with the key used to generate the strong name for Assembly A. If the .NET Framework security checks pass and the bind succeeds, Assembly B has a guarantee that Assembly A's bits have not been tampered with and that these bits actually come from the developers of Assembly A.</span></span>

> [!NOTE]
> <span data-ttu-id="aa17c-124">Questo scenario non permette di risolvere i problemi di attendibilità.</span><span class="sxs-lookup"><span data-stu-id="aa17c-124">This scenario doesn't address trust issues.</span></span> <span data-ttu-id="aa17c-125">Gli assembly possono contenere firme Microsoft Authenticode complete oltre a un nome sicuro.</span><span class="sxs-lookup"><span data-stu-id="aa17c-125">Assemblies can carry full Microsoft Authenticode signatures in addition to a strong name.</span></span> <span data-ttu-id="aa17c-126">Le firme Authenticode includono un certificato che definisce l'attendibilità.</span><span class="sxs-lookup"><span data-stu-id="aa17c-126">Authenticode signatures include a certificate that establishes trust.</span></span> <span data-ttu-id="aa17c-127">È importante tenere presente che i nomi sicuri non richiedono che il codice sia firmato in questo modo.</span><span class="sxs-lookup"><span data-stu-id="aa17c-127">It's important to note that strong names don't require code to be signed in this way.</span></span> <span data-ttu-id="aa17c-128">I nomi sicuri forniscono solo un'identità univoca.</span><span class="sxs-lookup"><span data-stu-id="aa17c-128">Strong names only provide a unique identity.</span></span>

- <span data-ttu-id="aa17c-135">È completamente attendibile senza prova <xref:System.Security.Policy.StrongName> (ad esempio, include la prova della zona `MyComputer`).</span><span class="sxs-lookup"><span data-stu-id="aa17c-135">Fully trusted without <xref:System.Security.Policy.StrongName> evidence (for example, has `MyComputer` zone evidence).</span></span>

- <span data-ttu-id="aa17c-136">Viene caricato in un dominio <xref:System.AppDomain> completamente attendibile.</span><span class="sxs-lookup"><span data-stu-id="aa17c-136">Loaded into a fully trusted <xref:System.AppDomain>.</span></span>

- <span data-ttu-id="aa17c-137">Viene caricato da una località nell'ambito della proprietà <xref:System.AppDomainSetup.ApplicationBase%2A> di <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="aa17c-137">Loaded from a location under the <xref:System.AppDomainSetup.ApplicationBase%2A> property of that <xref:System.AppDomain>.</span></span>

- <span data-ttu-id="aa17c-138">Non ha firma ritardata.</span><span class="sxs-lookup"><span data-stu-id="aa17c-138">Not delay-signed.</span></span>

<span data-ttu-id="aa17c-139">Questa funzionalità può essere disabilitata per singole applicazioni o per un computer.</span><span class="sxs-lookup"><span data-stu-id="aa17c-139">This feature can be disabled for individual applications or for a computer.</span></span> <span data-ttu-id="aa17c-140">Vedere [Procedura: disattivare la funzionalità di bypass](disable-strong-name-bypass-feature.md)con nome sicuro .</span><span class="sxs-lookup"><span data-stu-id="aa17c-140">See [How to: Disable the strong-name bypass feature](disable-strong-name-bypass-feature.md).</span></span>
