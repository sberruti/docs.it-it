<span data-ttu-id="4477d-149">L'approccio seguente è la più potente e potenzialmente pericolosa (se eseguita in modo non corretto) per la codifica di sicurezza: la libreria funge da interfaccia per il codice di accesso a determinate risorse che non sono altrimenti disponibili, così come le classi .NET applicano autorizzazioni per le risorse utilizzate.</span><span class="sxs-lookup"><span data-stu-id="4477d-149">The following approach is the most powerful and hence potentially dangerous (if done incorrectly) for security coding: your library serves as an interface for other code to access certain resources that aren't otherwise available, just as the .NET classes enforce permissions for the resources they use.</span></span> <span data-ttu-id="4477d-150">Se si espone una risorsa, è necessario per prima cosa esigere tramite codice l'autorizzazione adeguata alla risorsa (in altre parole, eseguire un controllo di sicurezza) e quindi eseguire un'asserzione dei relativi diritti per eseguire l'operazione vera e propria.</span><span class="sxs-lookup"><span data-stu-id="4477d-150">Wherever you expose a resource, your code must first demand the permission appropriate to the resource (that is, it must perform a security check) and then typically assert its rights to perform the actual operation.</span></span>
