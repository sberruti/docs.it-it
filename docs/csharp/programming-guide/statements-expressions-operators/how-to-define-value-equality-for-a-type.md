<span data-ttu-id="5e8e0-103">Quando si definisce una classe o uno struct, si decide se è opportuno creare una definizione personalizzata di uguaglianza di valore, o equivalenza, per il tipo.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="5e8e0-104">In genere, l'uguaglianza di valori viene implementata quando si prevede che oggetti del tipo vengano aggiunti a una raccolta, o quando lo scopo principale di tali oggetti consiste nell'archiviare un set di campi o di proprietà.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="5e8e0-105">È possibile basare la definizione di uguaglianza di valori su un confronto di tutti i campi e di tutte le proprietà nel tipo, oppure su un sottoinsieme.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span> <span data-ttu-id="5e8e0-106">Ma in entrambi i casi e in entrambe le classi e gli struct, l'implementazione deve seguire le cinque garanzie di equivalenza:</span><span class="sxs-lookup"><span data-stu-id="5e8e0-106">But in either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence:</span></span>  
  
1. <span data-ttu-id="5e8e0-107">`x.Equals(x)` restituisce `true`.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-107">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="5e8e0-108">Questa viene denominata proprietà riflessiva.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-108">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="5e8e0-109">`y.Equals(x)` restituisce lo stesso valore di `x.Equals(y)`.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-109">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="5e8e0-110">Questa viene denominata proprietà simmetrica.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-110">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="5e8e0-111">Se `(x.Equals(y) && y.Equals(z))` restituisce `true`, `x.Equals(z)` restituirà `true`.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-111">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="5e8e0-112">Questa viene denominata proprietà transitiva.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-112">This is called the transitive property.</span></span>

 <span data-ttu-id="5e8e0-116">Gli struct definiti hanno già un'implementazione predefinita di uguaglianza di valore che eredita dall'override <xref:System.ValueType?displayProperty=nameWithType> del metodo <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-116">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="5e8e0-117">Questa implementazione usa il processo di reflection per esaminare tutti i campi e tutte le proprietà nel tipo.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-117">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="5e8e0-118">Sebbene questa implementazione produca risultati corretti, è relativamente lenta rispetto a un'implementazione personalizzata che viene scritta specificamente per il tipo.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-118">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="5e8e0-119">I dettagli di implementazione per l'uguaglianza di valori sono diversi per le classi e gli struct.</span><span class="sxs-lookup"><span data-stu-id="5e8e0-119">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="5e8e0-120">Tuttavia, sia le classi che gli struct richiedono gli stessi passaggi di base per l'implementazione dell'uguaglianza:</span><span class="sxs-lookup"><span data-stu-id="5e8e0-120">However, both classes and structs require the same basic steps for implementing equality:</span></span>
