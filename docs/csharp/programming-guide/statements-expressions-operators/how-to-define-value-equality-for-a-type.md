<span data-ttu-id="f4d4d-103">Quando si definisce una classe o uno struct, si decide se è opportuno creare una definizione personalizzata di uguaglianza di valore, o equivalenza, per il tipo.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="f4d4d-104">In genere, l'uguaglianza di valori viene implementata quando si prevede che oggetti del tipo vengano aggiunti a una raccolta, o quando lo scopo principale di tali oggetti consiste nell'archiviare un set di campi o di proprietà.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="f4d4d-105">È possibile basare la definizione di uguaglianza di valori su un confronto di tutti i campi e di tutte le proprietà nel tipo, oppure su un sottoinsieme.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span> 

<span data-ttu-id="f4d4d-106">In entrambi i casi, e in entrambe le classi e gli struct, l'implementazione deve seguire le cinque garanzie di equivalenza (per le regole seguenti, si supponga che `x`, `y` e `z` non siano null):</span><span class="sxs-lookup"><span data-stu-id="f4d4d-106">In either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence (For the following rules, assume that `x`, `y` and `z` are not null):</span></span>  
  
1. <span data-ttu-id="f4d4d-107">`x.Equals(x)` restituisce `true`.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-107">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="f4d4d-108">Questa viene denominata proprietà riflessiva.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-108">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="f4d4d-109">`x.Equals(y)` restituisce lo stesso valore di `y.Equals(x)`.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-109">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="f4d4d-110">Questa viene denominata proprietà simmetrica.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-110">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="f4d4d-111">Se `(x.Equals(y) && y.Equals(z))` restituisce `true`, `x.Equals(z)` restituirà `true`.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-111">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="f4d4d-112">Questa viene denominata proprietà transitiva.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-112">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="f4d4d-113">Le successive chiamate di `x.Equals(y)` restituiscono lo stesso valore purché gli oggetti a cui x e y fanno riferimento non vengano modificati.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-113">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="f4d4d-114">Qualsiasi valore non null è diverso da null.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-114">Any non-null value is not equal to null.</span></span> <span data-ttu-id="f4d4d-115">Tuttavia, CLR verifica la presenza di valori null in tutte le chiamate al metodo e genera un'`NullReferenceException` se il riferimento `this` sarebbe null.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-115">However, the CLR checks for null on all method calls and throws a `NullReferenceException` if the `this` reference would be null.</span></span> <span data-ttu-id="f4d4d-116">Pertanto, `x.Equals(y)` genera un'eccezione quando `x` è null.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-116">Therefore, `x.Equals(y)` throws an exception when `x` is null.</span></span> <span data-ttu-id="f4d4d-117">Che interrompe le regole 1 o 2, a seconda dell'argomento `Equals`.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-117">That breaks rules 1 or 2, depending on the argument to `Equals`.</span></span>
 
 <span data-ttu-id="f4d4d-118">Gli struct definiti hanno già un'implementazione predefinita di uguaglianza di valore che eredita dall'override <xref:System.ValueType?displayProperty=nameWithType> del metodo <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-118">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="f4d4d-119">Questa implementazione usa il processo di reflection per esaminare tutti i campi e tutte le proprietà nel tipo.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-119">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="f4d4d-120">Sebbene questa implementazione produca risultati corretti, è relativamente lenta rispetto a un'implementazione personalizzata che viene scritta specificamente per il tipo.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-120">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="f4d4d-121">I dettagli di implementazione per l'uguaglianza di valori sono diversi per le classi e gli struct.</span><span class="sxs-lookup"><span data-stu-id="f4d4d-121">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="f4d4d-122">Tuttavia, sia le classi che gli struct richiedono gli stessi passaggi di base per l'implementazione dell'uguaglianza:</span><span class="sxs-lookup"><span data-stu-id="f4d4d-122">However, both classes and structs require the same basic steps for implementing equality:</span></span>
